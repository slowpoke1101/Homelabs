1. Switch the compositor to XRender (lighter than OpenGL)
OpenGL looks nice, but VMware’s virtual GPU doesn’t accelerate it well.
- System Settings
- Display and Monitor
- Compositor
- Rendering backend → XRender
- Apply
2. Disable heavy animations
Still in Compositor:
- Animation speed → Fast
- Uncheck:
- Blur
- Wobbly windows
- Sliding popups
- Dim inactive windows
- Background contrast
These effects look cool but eat CPU cycles in a VM.

3. Disable desktop effects you don’t need
System Settings → Workspace Behavior → Desktop Effects
Turn off:
- Blur
- Fade desktop
- Magic lamp
- Scale
- Sliding windows
- Translucency
- Window open/close animations
Keep:
- Present windows (optional)
- Overview (optional)
- Screen edge effects (optional)
This keeps KDE feeling like KDE, just lighter.

4. Reduce background services
KDE starts a lot of helpers you don’t need in a lab VM.
System Settings → Startup and Shutdown → Background Services
Disable:
- KDE Connect
- Printer notifier
- Bluetooth
- PowerDevil (optional if on desktop VM)
- Baloo file indexer (see next step)
5. Disable Baloo file indexing
Baloo is KDE’s search indexer — great on real hardware, heavy in a VM.
Run:
balooctl disable
This stops background disk scanning and CPU spikes.
6. Disable KDE Wallet (optional)
If you don’t use it:
System Settings → KDE Wallet
Uncheck “Enable the KDE wallet subsystem”
This removes annoying prompts and background processes.
7. Set a lighter window decoration
System Settings → Appearance → Window Decorations
Choose:
- Breeze (default)
or
- Breeze Light
Avoid the more animated themes.
8. Disable desktop search
System Settings → Search → File Search
Turn off:
- Enable File Search
- Enable Content Search
This prevents background indexing.
9. Reduce swappiness
VMs benefit from less swapping.
echo "vm.swappiness=10" | sudo tee -a /etc/sysctl.conf
